# CLAUDE.md

このファイルは、このリポジトリで Claude Code が作業する際に厳守すべきルールを定義する。

## Git / GitHub ワークフロー

このリポジトリでは以下のワークフローを**例外なく**遵守すること。ユーザーが明示的に簡略化を指示した場合を除き、省略しない。

### 1. すべての作業は GitHub Issue から始める

実装作業に着手する前に、必ず `gh issue create` で Issue を作成する。Issue には以下を含める。

- 背景・目的（なぜその変更が必要か）
- 対応内容の概要
- 完了条件

トリアージ的な調査や質問への回答など、コードの変更を伴わない作業では Issue 作成は不要。

### 2. ブランチ命名規則

`main` から最新の状態で分岐し、以下の形式で命名する。

```
<type>/<issue番号>-<kebab-caseの説明>
```

- `type` は次のいずれか: `feature` / `fix` / `docs` / `refactor` / `chore` / `test`
- 例: `feature/12-add-login`, `fix/15-null-error`

### 3. main への直接コミット・push は禁止

`main` ブランチへの直接コミット・直接 push は行わない（GitHub 側のブランチ保護でも禁止されている）。必ず上記のブランチ上で作業し、`gh pr create` で PR を作成する。PR 本文には `Closes #<issue番号>` を含め、対応する Issue と紐付けること。

### 4. PR のマージはユーザーの承認を得てから

PR の作成までは自律的に行ってよいが、`main` へのマージはユーザーに明示的に確認を取ってから実行する。

### 5. 例外

軽微な修正などでユーザーが「Issue は不要」「このまま直接直して」等、明示的に簡略化を指示した場合はその指示に従ってよい。デフォルトの挙動は上記の厳格なフローとする。
