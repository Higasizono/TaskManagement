# 要件定義書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | タスク管理アプリ（Trello風） |
| 目的 | 個人のタスクをボード・カラム・カードで視覚的に管理する |
| 対象ユーザー | 自分1人（スクール課題） |
| 利用環境 | Webブラウザ（PC） |

---

## 2. 機能要件

### 2-1. ボード管理

| # | 機能 | 詳細 |
|---|------|------|
| B-01 | ボード一覧表示 | トップページにボードのカード一覧を表示する |
| B-02 | ボード作成 | ボード名を入力して新規ボードを作成できる |
| B-03 | ボード削除 | ボードを削除できる（配下のカラム・カードも全て削除される） |
| B-04 | ボード選択 | ボードをクリックするとボード詳細ページに遷移する |

### 2-2. カラム管理

| # | 機能 | 詳細 |
|---|------|------|
| C-01 | カラム表示 | ボード内のカラムを横並びで表示する |
| C-02 | カラム追加 | カラム名を入力して新しいカラムをボードに追加できる |
| C-03 | カラム削除 | カラムを削除できる（配下のカードも全て削除される） |

### 2-3. カード管理

| # | 機能 | 詳細 |
|---|------|------|
| K-01 | カード表示 | カラム内のカードを縦に一覧表示する |
| K-02 | カード追加 | タイトルを入力してカラムにカードを追加できる |
| K-03 | カード編集 | カードのタイトルをクリックして編集できる |
| K-04 | カード削除 | カードを削除できる |
| K-05 | カード移動 | ドラッグ&ドロップでカードを別カラムに移動できる |

### 2-4. データ保存

| # | 機能 | 詳細 |
|---|------|------|
| D-01 | 自動保存 | 操作のたびにlocalStorageへ自動保存する |
| D-02 | データ復元 | ページ読み込み時にlocalStorageからデータを復元する |
| D-03 | 初期データ | 初回起動時はデフォルトのボードとカラムを表示する |

---

## 3. 非機能要件

| 項目 | 内容 |
|------|------|
| 認証 | 不要（ログインなし） |
| レスポンシブ | PCブラウザのみ対応（スマホは対象外） |
| データ保存先 | ブラウザのlocalStorage（サーバー不要） |
| パフォーマンス | 特別な要件なし（個人利用のため） |

## 4. 画面構成

### 4-1. 画面一覧

| 画面名 | URL | 説明 |
|--------|-----|------|
| トップページ | `/` | ボード一覧を表示 |
| ボードページ | `/board/:id` | カラムとカードを表示 |

## 5. データ設計

### 5-1. データ構造

```json
{
  "boards": [
    {
      "id": "board-1",
      "title": "スクール課題",
      "createdAt": "2026-06-27T10:00:00"
    }
  ],
  "columns": [
    { "id": "col-1", "boardId": "board-1", "title": "未着手", "order": 0 },
    { "id": "col-2", "boardId": "board-1", "title": "進行中", "order": 1 },
    { "id": "col-3", "boardId": "board-1", "title": "完了",   "order": 2 }
  ],
  "cards": [
    {
      "id": "card-1",
      "columnId": "col-1",
      "title": "デザインを考える",
      "order": 0,
      "createdAt": "2026-06-27T10:00:00"
    }
  ]
}
```

### 5-2. フィールド定義

**boards**

| フィールド | 型 | 必須 | 説明 |
|------------|----|------|------|
| id | string | ○ | 一意のID（UUID） |
| title | string | ○ | ボード名 |
| createdAt | string | ○ | 作成日時（ISO 8601） |

**columns**

| フィールド | 型 | 必須 | 説明 |
|------------|----|------|------|
| id | string | ○ | 一意のID（UUID） |
| boardId | string | ○ | 所属するボードのID |
| title | string | ○ | カラム名 |
| order | number | ○ | 表示順序（0始まり） |

**cards**

| フィールド | 型 | 必須 | 説明 |
|------------|----|------|------|
| id | string | ○ | 一意のID（UUID） |
| columnId | string | ○ | 所属するカラムのID |
| title | string | ○ | タスク名 |
| order | number | ○ | カラム内の表示順序（0始まり） |
| createdAt | string | ○ | 作成日時（ISO 8601） |

## 6. 技術スタック

| 項目 | 選定技術 | 理由 |
|------|----------|------|
| フレームワーク | React + Vite | セットアップが速い・バックエンド不要 |
| 言語 | TypeScript | 型安全でデータ設計がしやすい |
| スタイル | CSS Modules または Tailwind CSS | 未定 |
| ドラッグ&ドロップ | dnd-kit | React向けの軽量ライブラリ |
| データ保存 | localStorage | サーバー不要・ブラウザ内で完結 |
| ID生成 | crypto.randomUUID() | ブラウザ標準API・ライブラリ不要 |

---

## 7. 今回のスコープ外

以下は今回実装しない。

- ユーザー認証・ログイン
- カードへのメモ・期限・担当者
- カラムの並び替え（ドラッグ）
- スマートフォン対応
- サーバー・データベース連携
