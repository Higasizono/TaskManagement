## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | タスク管理アプリ（Trello風） |
| 背景・目的 | Reactを用いたフロントエンド開発の学習を目的としたスクール課題。コンポーネント設計・状態管理・ドラッグ&ドロップ・ローカルストレージを用いたデータ永続化を実践的に習得する |
| 学習目標 | ① Reactコンポーネントの分割設計　② TypeScriptによる型安全な実装　③ ドラッグ&ドロップUIの実装　④ localStorageを使ったデータ永続化 |
| 対象ユーザー | 本人のみ（スクール課題のため外部公開なし） |
| 利用環境 | Webブラウザ（PCのみ） |

## 2. 機能要件

### 2-1. ボード管理

| # | 機能 | 詳細 |
|---|------|------|
| B-01 | ボード一覧表示 | トップページにボードのカード一覧を表示する |
| B-02 | ボード作成 | ボード名を入力して新規ボードを作成できる |
| B-03 | ボード削除 | ボードを削除できる（配下のカラム・カードも全て削除される） |
| B-04 | ボード選択 | ボードをクリックするとボード詳細ページに遷移する |

### 2-2. カラム管理

| # | 機能 | 詳細 |
|---|------|------|
| C-01 | カラム表示 | ボード内のカラムを横並びで表示する |
| C-02 | カラム追加 | カラム名を入力して新しいカラムをボードに追加できる |
| C-03 | カラム削除 | カラムを削除できる（配下のカードも全て削除される） |

### 2-3. カード管理

| # | 機能 | 詳細 |
|---|------|------|
| K-01 | カード表示 | カラム内のカードを縦に一覧表示する |
| K-02 | カード追加 | タイトルを入力してカラムにカードを追加できる |
| K-03 | カード編集 | カードのタイトルをクリックして編集できる |
| K-04 | カード削除 | カードを削除できる |
| K-05 | カード移動 | ドラッグ&ドロップでカードを別カラムに移動できる |

### 2-4. データ保存

| # | 機能 | 詳細 |
|---|------|------|
| D-01 | 自動保存 | 操作のたびにlocalStorageへ自動保存する |
| D-02 | データ復元 | ページ読み込み時にlocalStorageからデータを復元する |
| D-03 | 初期データ | 初回起動時はデフォルトのボードとカラムを表示する |

## 3. 非機能要件

### 3-1. 動作環境

| 項目 | 内容 |
|------|------|
| 対応ブラウザ | Google Chrome（最新版）、Safari（最新版）、Firefox（最新版） |
| 最小画面幅 | 1024px以上（それ以下は表示崩れが起きても許容） |
| レスポンシブ | 非対応（スマートフォン・タブレットは対象外） |

### 3-2. パフォーマンス

| 項目 | 内容 |
|------|------|
| UI応答性 | カード追加・削除・移動などの操作が視覚的に即時反映されること（ローディングなし） |
| ドラッグ操作 | ドラッグ中のアニメーションがカクつかないこと（60fps目安） |
| 初期表示 | ページ読み込み後、1秒以内にコンテンツが表示されること |

### 3-3. データ・セキュリティ

| 項目 | 内容 |
|------|------|
| データ保存先 | ブラウザのlocalStorage（サーバー・DBなし） |
| データ永続性 | ブラウザを閉じて再度開いてもデータが維持されること |
| XSS対策 | ユーザー入力はReactの標準エスケープ機構により画面上に安全に出力されること |
| 認証 | 不要（ローカル環境での個人利用のみ） |

### 3-4. コード品質・保守性

| 項目 | 内容 |
|------|------|
| 型チェック | TypeScript の `strict` モードを有効にする |
| コンポーネント設計 | 1コンポーネント1責務を意識した分割を行う |
| ファイル構成 | 機能・役割ごとにディレクトリを整理する |

### 3-5. アクセシビリティ

| 項目 | 内容 |
|------|------|
| キーボード操作 | ボード・カラム・カードの作成・削除がキーボードのみでも操作できること |
| フォーカス制御 | 入力フィールドが表示されたとき、自動的にフォーカスが当たること |

## 4. 画面構成

### 4-1. 画面一覧と遷移図

```
トップページ (/)
    │
    │ ボードをクリック
    ▼
ボードページ (/board/:id)
    │
    │ 「← 戻る」をクリック
    ▼
トップページ (/)
```

| 画面名 | URL | 説明 |
|--------|-----|------|
| トップページ | `/` | ボード一覧を表示・作成・削除 |
| ボードページ | `/board/:id` | カラムとカードを表示・操作 |

**レイアウト**

```
┌─────────────────────────────────────────────┐
│  タスク管理アプリ                             │  ← ヘッダー（アプリ名）
├─────────────────────────────────────────────┤
│                                             │

│                                             │

```
┌──────────────────┐
│ ボードA        × │  ← ボード名 + 削除ボタン
└──────────────────┘
```

**空状態（ボードが0件のとき）**

```
┌─────────────────────────────────────────────┐
│  タスク管理アプリ                             │
├─────────────────────────────────────────────┤
│                                             │

**ボード作成時のインライン入力**

## 5. ユースケース / 操作フロー

### UC-01: ボードを作成する

- **前提条件**: トップページが表示されている
- **操作手順**:
  1. 「＋ 新規ボードを作成」ボタンをクリックする
  2. インライン入力フィールドが表示され、自動的にフォーカスが当たる
  3. ボード名を入力する
  4. Enterキーを押す、または「作成」ボタンをクリックする
  5. 新しいボードがボード一覧の末尾に追加される
- **代替フロー（キャンセル）**: 手順3でEscキーを押すと入力フィールドが閉じ、ボードは作成されない
- **エラー時**: バリデーションエラーの場合はエラーメッセージを入力フィールド付近に表示し、ボードは作成されない

---

### UC-02: ボードを削除する

- **前提条件**: トップページにボードが1件以上表示されている
- **操作手順**:
  1. 削除したいボードカードの「×」ボタンをクリックする
  2. 確認ダイアログを表示する（「本当に削除しますか？配下のカラム・カードもすべて削除されます」）
  3. 「削除する」をクリックする
  4. ボードがリストから削除される
- **代替フロー（キャンセル）**: 手順3で「キャンセル」をクリックすると削除されない

---

### UC-03: カラムを追加する

- **前提条件**: ボードページが表示されている
- **操作手順**:
  1. 「＋ カラムを追加」ボタンをクリックする
  2. インライン入力フィールドが表示され、フォーカスが当たる
  3. カラム名を入力する
  4. Enterキーを押す、または「追加」ボタンをクリックする
  5. 新しいカラムが一覧の末尾に追加される
- **代替フロー**: Escキーでキャンセル

---

### UC-04: カラムを削除する

- **前提条件**: ボードページにカラムが1件以上表示されている
- **操作手順**:
  1. 削除したいカラムの「×」ボタンをクリックする
  2. 確認ダイアログを表示する（「このカラム内のカードもすべて削除されます」）
  3. 「削除する」をクリックする
  4. カラムと配下のカードが削除される

---

### UC-05: カードを追加する

- **前提条件**: ボードページにカラムが1件以上表示されている
- **操作手順**:
  1. カードを追加したいカラムの「＋ カードを追加」をクリックする
  2. カラム内にインライン入力フォームが展開され、フォーカスが当たる
  3. カードのタイトルを入力する
  4. Enterキーを押す、または「追加」ボタンをクリックする
  5. 新しいカードがカラムの末尾に追加される
- **代替フロー**: Escキーまたは「キャンセル」でフォームが閉じる

---

### UC-06: カードを編集する

- **前提条件**: ボードページにカードが1件以上表示されている
- **操作手順**:
  1. 編集したいカードのタイトルをクリックする
  2. タイトルがインライン入力フィールドに切り替わり、現在のテキストが選択された状態になる
  3. テキストを編集する
  4. Enterキーを押して確定する
  5. カードのタイトルが更新される
- **代替フロー**: Escキーで編集をキャンセルし、元のタイトルに戻る
- **エラー時**: バリデーションエラーの場合はエラーメッセージを表示し、変更は保存されない

---

### UC-07: カードを削除する

- **前提条件**: ボードページにカードが1件以上表示されている
- **操作手順**:
  1. 削除したいカードの「×」ボタンをクリックする
  2. カードが即時削除される（確認ダイアログなし）

---

### UC-08: カードをドラッグ&ドロップで移動する

- **前提条件**: ボードページにカードが1件以上、カラムが2件以上表示されている
- **操作手順**:
  1. 移動したいカードをマウスでドラッグする
  2. ドラッグ中、カードがゴースト表示になり、移動先のカラムがハイライトされる
  3. 移動先のカラムにドロップする
  4. カードが移動先カラムの指定位置に追加される
- **代替フロー**: 元の位置に戻したい場合はドロップせずに元の場所に戻すか、ドラッグをキャンセルする（ESCキー）

## 6. バリデーションルール

入力フォームすべてに以下のルールを適用する。

| 対象 | フィールド | ルール | エラーメッセージ |
|------|-----------|--------|-----------------|
| ボード | タイトル | 必須・最大50文字・前後の空白は自動除去 | 「ボード名を入力してください」 |
| カラム | タイトル | 必須・最大30文字・前後の空白は自動除去 | 「カラム名を入力してください」 |
| カード | タイトル | 必須・最大100文字・前後の空白は自動除去 | 「カード名を入力してください」 |
| 全共通 | - | 空白文字のみの入力は無効（trim後に長さ0になる場合） | 各「入力してください」メッセージを流用 |

**エラー表示方針**

- エラーメッセージは入力フィールドの直下に赤文字で表示する
- 入力内容が修正された時点でエラーメッセージは消える
- エラー状態の間は作成・保存ボタンを非活性にしない（送信時に再チェックする）

## 7. データ設計

### 7-1. データ構造

```json
{
  "boards": [
    {
      "id": "board-1",
      "title": "スクール課題",
      "createdAt": "2026-06-27T10:00:00"
    }
  ],
  "columns": [
    { "id": "col-1", "boardId": "board-1", "title": "未着手", "order": 0 },
    { "id": "col-2", "boardId": "board-1", "title": "進行中", "order": 1 },
    { "id": "col-3", "boardId": "board-1", "title": "完了",   "order": 2 }
  ],
  "cards": [
    {
      "id": "card-1",
      "columnId": "col-1",
      "title": "デザインを考える",
      "order": 0,
      "createdAt": "2026-06-27T10:00:00"
    }
  ]
}
```

### 7-2. フィールド定義

**boards**

| フィールド | 型 | 必須 | 説明 |
|------------|----|------|------|
| id | string | ○ | 一意のID（UUID） |
| title | string | ○ | ボード名（最大50文字） |
| createdAt | string | ○ | 作成日時（ISO 8601） |

**columns**

| フィールド | 型 | 必須 | 説明 |
|------------|----|------|------|
| id | string | ○ | 一意のID（UUID） |
| boardId | string | ○ | 所属するボードのID |
| title | string | ○ | カラム名（最大30文字） |
| order | number | ○ | 表示順序（0始まり） |

**cards**

| フィールド | 型 | 必須 | 説明 |
|------------|----|------|------|
| id | string | ○ | 一意のID（UUID） |
| columnId | string | ○ | 所属するカラムのID |
| title | string | ○ | タスク名（最大100文字） |
| order | number | ○ | カラム内の表示順序（0始まり） |
| createdAt | string | ○ | 作成日時（ISO 8601） |

## 8. 技術スタック・選定理由

| レイヤー | 採用技術 | バージョン | 選定理由 |
|----------|----------|-----------|----------|
| UIフレームワーク | React | 18.x | コンポーネントベースの設計を学ぶ。学習目標に直結するため選定 |
| ビルドツール | Vite | 5.x | セットアップが速く、HMRによる開発体験が良い。バックエンド不要 |
| 言語 | TypeScript | 5.x | 型安全なコードを書く習慣を身につける。データ設計の明示にも有効 |
| スタイリング | Tailwind CSS | 3.x | クラス名でスタイルを完結させ、CSS設計の複雑さを排除。学習コストが低い |
| ドラッグ&ドロップ | dnd-kit | 6.x | React向けの軽量ライブラリ。アクセシビリティ対応済み・APIがシンプル |
| データ保存 | localStorage（ブラウザ標準） | — | サーバー・DB不要でブラウザ内完結。課題スコープに適している |
| ID生成 | crypto.randomUUID()（ブラウザ標準） | — | 追加ライブラリ不要。モダンブラウザで広くサポートされている |
| ルーティング | React Router | 6.x | SPAの画面遷移（ボード一覧 ↔ ボード詳細）を実装するために使用 |

## 9. 今回のスコープ外

以下は今回実装しない。

- ユーザー認証・ログイン
- カードへのメモ・期限・担当者
- カラムの並び替え（ドラッグ）
- スマートフォン・タブレット対応
- サーバー・データベース連携
- ボード・カラムの名前変更
- カードの検索・フィルタリング
